Everything you need, nothing you don't

A purpose-built terminal for mobile. Every feature exists because typing on a phone is painful.

[ ]

Smart Buttons

Auto-detects interactive prompts (Y/n, Allow/Deny, numbered options) and renders one-tap buttons. No more typing on tiny keyboards.

>_

Claude Code Ready

Built-in support for Claude Code CLI. Approve, deny, or auto-approve AI actions with safety checks for dangerous commands.

|||

Multi-Tab Sessions

Up to 4 concurrent tmux sessions. Each tab survives disconnects — pick up where you left off.

))

Voice Input

Dictate commands with push-to-talk. Browser Speech API with optional Azure Whisper server-side fallback.

#

Telegram-Native Auth

HMAC-SHA256 verification via Telegram. No passwords, no SSH keys to manage. Whitelist by user ID.

!

Emergency Access

Management API lets you execute commands via HTTPS when SSH is down. Your lifeline when nothing else works.

Three steps to terminal access

From Telegram to a running terminal in seconds. No setup on your phone.

1

Open Telegram

Tap the menu button on your teletty bot. The terminal loads instantly as a Mini App.

2

Connect

Telegram authenticates you automatically. No passwords, no keys. If you're whitelisted, you're in.

3

Run anything

Full terminal with tmux persistence. Smart buttons appear for interactive prompts. Voice input for hands-free operation.

Up and running in 60 seconds

Three ways to deploy. Pick your favorite.

bash 
npx teletty init
# Edit .env with your BOT_TOKEN and ALLOWED_USER_IDS

bash

cp .env.example .env && nano .env
docker compose up -d
bash

Simple by design

No build step. No frameworks. Just Node.js and a WebSocket.

Telegram (tap Menu Button)
  -> Mini App WebView
  -> POST /auth (HMAC-SHA256)
  -> WebSocket (wss://)
  -> node-pty -> tmux
  -> xterm.js renders output
  -> output-parser -> smart buttons

889 lines of code. 7 files. Zero build step. Zero frameworks.
